Darts Club Management System

Overview

I’ve created my project with the goal of attempting to improve the world of darts through the creation of a Darts Club Management System (DCMS), the DCMS is created through combining modern web and app technology. The DCSM will do away with non-random competition draws, chalkboard scoring, mistaken mathematics, and, provide a live stream of any current darts game dart-by-dart, whilst also linking the clubs around the world statistically, and socially.

Motivation

I love darts, I love the social aspect, the competitive aspect, and, the history and heritage of the game. What I don’t love is how it so isolated to you and your team, extending only really to the local clubs and not much further. For years world darts leagues are ‘chalked’ ie they’re scored on a chalkboard. Once the game is over there’s a piece of paper with the scores and a few stats but nothing permanent – occasional darts websites excepted. There’s also nothing to tie the clubs together, or the leagues, or the greater world darts community. There’s no standardised club management system despite a (mostly) standard set of club rules, there’s no standard way to draw players and have them compete in a league without ‘borrowing’ from others websites. There’s no statistics unless you’re really good, but no one truly knows you’re really good unless you turn up and go ‘through the ranks’ and hope local politics doesn’t have you sidelined before you’ve warmed up.

Until now.

Description

At it’s most basic funciton, the DCMS will allow for members to register and select a club they are a member of. The Club can then manage the membership of individual members. Expanding on this will be the management of leagues, matches and game draws; thse would be able to be completed either online or using a mobile app. Then, at it’s most advanced stage, the DCMS can be used for scoring each individual game, and tracking each dart thrown to be able to compile and present statitics for each member, club, league, competition and tournament. An example of the statistics offered may be player accuracy heat maps - which can then be used to provide fair matchmaking or handicaps. Included in the DCMS will be varying rulesets for each game, turnkey rulesets, charters, and constitutions for each club, committee management tools, member voting tools, the ability to connect to other players, match broadcasting tools either through a weblink to a match-in-progress where scores are updated in real time or by connection through Twitter API and auto-tweeting games as they’re played.

Tools and Technologies

A web based framework woud be required ie Django or Ruby on Rails, the framework would require a database such as PostGreSQL. The framework would be required to support an Application Programming Interface (API) to enable communication to the database and backend from the webs frontend and the mobile apps. The web frontend would be created using the react.js framework, the Android and Apple interface will be created using Java and Swift respectively. JavaScript will be used for the smart TV app. Version Control System (VCS) will be via Bitbucket, the chosen IDE(s) will be from the JetBrains toolbox. Hosting of the online requirements (frontend, backend, database) will be via Amazon Web Services (AWS).

Skills Required

Although the skills required for this would seem broad at first, they are mostly ‘ordinary web building skills.

  • Django or Ruby on Rails
  • PostGreSQL
  • Swift
  • Java,
  • JavaScript, including react.js
  • VCS / Git.
  • AWS server management
No special hardware would be required. The software skills for creating this are easily found in online marketplaces such as Freelancer. From my experience the most difficult part is deploying to the AWS environment. This is due to expertise required in establishing scalable cloud infrastrure, and, the lack of tutorials an duser guides available – which provide limited help due to not being maintained as fast as the AWS environ,ent changes. A person with AWS experience would be required to be found.

Outcome

If the project is successful it will provide an alternative to the current non-harmonised darts clubs, provide structure and uniformity to darts club management, and, enable smaller clubs to run at the same level as larger clubs without labourious administrative overheads, due to it’s online club management framework. Whether or not this project has a real world impact would be decided upon by the club’s own uptake of the system.
